Running LiquiBase

There are five main ways to run LiquiBase:

  • As an Ant task
  • As a Maven plugin
  • As part of a Spring-based application
  • As part of a Grails-based application
  • As part of an application server startup
  • Through a command line application

All four ways have various strengths and weaknesses. The Ant/Maven runners work well for integrating changes into an existing build process. The Spring migrator and servlet listener works well for making sure that a database is up-to-date each time a site is deployed. The command line tool works well outside a build environment when you want to do on-demand migrations of a database without any dependencies, or if you want to use LiquiBase's advanced execution options such as saving the migration SQL or controlling the changelog database lock.

If none of the above ways work for you, or if you would like to integrate LiquiBase into your application, the primary front-end class is liquibase.migrator.Migrator
