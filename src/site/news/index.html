LiquiBase News

5/23/2007 - LiquiBase 0.9.1 Released

LiquiBase 0.9.1 has been released. 0.9.1 is a bug-fix release that fixes bugs in the Ant migrator, the command line migrator, and the "modify column" refactoring.

5/14/2007 - LiquiBase 0.9.0 Released

LiquiBase 0.9.0 has been released. 0.9.0 is feature complete and the last major release before the 1.0 release candidate series. Between 0.9 and 1.0-RC1 we are going focus on bug fixes and finalizing public APIs. We also hope to have a beta visual LiquiBase editor based on the SQuirreL SQL client available with the 1.0-RC1 release. Other IDE plug-ins will become available after the 1.0 release.

Major changes in 0.9.0 include:

Roll-Back Support

We have added support for custom and auto-generated rollback statements. This allows you to bring your database back to the state it was at an earlier date. Read More

Maven Support

We have added beta support for Maven. Although we followed the documentation on the Maven site for creating a plug-in, we we do not have the necessary Maven experience to actually test it well. If you are interested in helping us test this functionality, please contact us.

New Refactorings

  • Create View
  • Rename View
  • Drop View
  • Merge Columns
  • Add Foreign-Key Constraint
  • Drop Foreign-Key Constraint
  • Add Lookup Table
  • Add Primary Key
  • Drop Primary Key
  • Add Auto-Increment
  • Add Default Value
  • Drop Default Value
  • Add Unique Constraint
  • Drop Unique Constraint

Command-line migrator cleanup

We have changed the format of options and commands passed to the command line migrator to make it more user friendly. We also removed logging by default, although it can be re-enabled with a flag. Read More

Ant changes

We changed the "migrationFiles" parameter changed to "changeLogFile" and no longer supports multiple change-logs.

DATABASECHANGELOG table changes

We added additional columns to better track changes. LiquiBase will update your DATABASECHANGLOG table automatically.

Other

As we approach 1.0, we are continuing to fix bugs, improve performance, and improve documentation.

4/25/2007 - LiquiBase 0.8.1 Released

0.8.1 fixes a bug related to running LiquiBase without specifing an execution context.

We also changed prompting on non-localhost databases to be "false" by default, but made it an option in the command-line migrator.

4/22/2007 - LiquiBase 0.8 Released

With version 0.8, we have finished the LiquiBase name change and completed our move to SourceForge. We have also added many useful new features including:

Change Log Preconditions

Preconditions can be specified at the beginning of each change log to both document and enforce assumptions the writers of the changelog had when creating it.

Currently supported preconditions check the type of database (MySQL vs. PostgreSQL) and the username of the database connection.

More on preconditions

Execution contexts

Change Sets can now be tagged with a "context" in which to run. This allows you to control which will be executed in any particular migration run.

Contexts can be used to ease management of test data sets and to allow you to more easily create a single change log that is compatible with multiple DBMS systems.

More on contexts

failOnError

ChangeSets can now have a "failOnError" attribute which controls if errors executing the changeset will cause the entire database migration to fail or not.

Add/Drop Not-Null Constraints

Two new refactorings have been added: "addNotNullConstraint" and "dropNotNullConstraint"

Comment tags in change sets

We have added a "comment" tag. Currently it is just ignored, but future releases may make use of it.

Improved documentation

We have officially launched http://www.liquibase.org with greatly improved documentation. The documentation is also included in the downloaded .zip files as well.

Many, Many Bug Fixes

As we approach 1.0, we have added an expanded set of unit and integration tests which have helped us flush out many bugs and will continue to make sure that old issues do not resurface.

We would like to thank Thilak Kumar Rajaraman, Ramachandra Kadam, and Yashwanth Potla for the contributions they have made to this version of LiquiBase.

3/16/2007 - LiquiBase 0.7 Released

We are proud to announce LiquiBase, the newly-renamed Sundog Database Refactoring Tool. We were never really fond of the SDRT name and finally found one we like better.

The 0.7 version is primarily focused around the name conversion. It has a new package structure based on liquibase.* rather than net.sundog.db.*. We are also working on updating our website and moving much of the project management information to http://sourceforge.net/projects/liquibase .

Beyond the name change, the following changes were made:

  • Added support for autoIncrement columns in createTableChange
  • Better support for multi-line SQL scripts

Note: DUE THE THE CHANGE IN MULTI-LINE SQL HANDLING, YOU MUST REBUILD YOUR MD5SUMS. Execute "update databasechangelog set md5sum=null" to clear out your past md5sums. They will be rebuilt with new correct new versions the next time your run the migrator.
