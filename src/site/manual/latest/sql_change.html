<sql> tag

The "sql" tag allows you to specify whatever sql you want. It is useful for complex changes that aren't supported through LiquiBase's automated refactoring tags and to work around bugs and limitations of LiquiBase.

Currently, the sql tag is the best way to create stored procedures and views. 

<sql>insert into person (id, name) values (1, 'Bob') </sql>
