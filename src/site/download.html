LiquiBase: Database Refactoring

@NAV.MAIN@

Download LiquiBase

The current version of the LiquiBase (was Sundog Database Refactoring Tool) is @BUILD.VERSION@. The XML changelog schema is ready for production use as any changes to the schema will be backwards compatible up to and through version 1.0.

Download Source And Binaries From SourceForge

Upgrade Notes:

  • Between 0.8.x and 0.9.0, the Ant task "migrationFiles" parameter changed to "changeLogFile" and no longer supports multiple change logs.
  • Between 0.8.0 and 0.8.1, LiquiBase no longer prompts you if you run it against a non-localhost database. There is a flag available in the command line migrator and the ant migrator that allows you to turn the functionality back on.
  • Between 0.7.0 and 0.8.0 the location and namespace of the XSD changed. The old XSD will remain available, but does not contain the most current schema
  • Between 0.6.2 and 0.7.0 there was a change to how MD5Sums were generated for changelogs. When upgrading, you will need to run update databasechangelog set md5sum=null to clear existing MD5Sums. They will be re-generated with the correct values on the next migration
  • Between 0.6.2 and 0.7.0 we began our transition our new "LiquiBase" name Package names changed which should have no impact on end-users beyond a change to the name of the ServletMigrator listener which is now liquibase.migrator.servlet.ServletMigrator.

Requirements

LiquiBase has been designed for Java 1.5 and greater. There is a package built for 1.4, but it has not been well tested.

Installation

Extract liquibase.zip. In it you will find a liquibase.migrator.jar This jar is all that is needed to run the migration tool from within ant or a servlet container. You can run the command line version of the migrator with the same jar file by simply using java -jar liquibase.migrator.jar.
